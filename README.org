* Sobre

Este repositório contém configurações /dotfiles/ para diversas ferramentas de IA — agentes de código, CLIs, dashboards e serviços Docker — todos versionados em um só lugar para facilitar sincronização entre máquinas.

* Estrutura do Repositório

│   ├── droid/                  # Factory.ai Droid
│   ├── hermes-workspace/       # Docker Compose para Hermes Agent
│   ├── homepage-navi/          # Homepage dashboard

* Configurações por Ferramenta

Cada subdiretório de =configs/= é uma fonte de verdade (/source of truth/) para as configurações de uma ferramenta. Sincronize com =cp= ou =ln -s= conforme preferir.

** sprachspiel — Sprachspiel CLI Agent

- **Ferramenta**: [[https://github.com/luksamuk/sprachspiel][Sprachspiel]] — agente de IA no terminal com RAG e ferramentas MCP.
- **Diretório**: =configs/sprachspiel/=
- **Sync**: Os configs são gerenciados internamente pelo Sprachspiel; este diretório serve como backup/versionamento.

** codex — OpenAI Codex CLI

- **Ferramenta**: [[https://github.com/openai/codex][OpenAI Codex CLI]] — agente de código no terminal (Codex 0.128+).
- **Diretório**: =configs/codex/=
- **Arquivos**: =config.toml= — perfis de modelo (Ollama Cloud, llama-swap local, Sisyphus Pi), projetos confiáveis.
- **Destino**: =~/.codex/config.toml=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/codex/config.toml ~/.codex/config.toml
  #+END_SRC

** crush — Charm Crush CLI

- **Ferramenta**: [[https://charm.land/crush][Charm Crush]] — CLI de chat com IA, suporta múltiplos providers e MCP.
- **Diretório**: =configs/crush/=
- **Arquivos**: =crush.json= — providers (Ollama Cloud, llama.cpp local, Sisyphus Pi), modelos, MCP servers (DuckDuckGo via uvx, SearXNG), LSP, permissões.
- **Destino**: =~/.local/share/crush/crush.json=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/crush/crush.json ~/.local/share/crush/crush.json
  #+END_SRC

** hermes-workspace — Docker

- **Ferramenta**: [[https://hermes-agent.nousresearch.com][Hermes Agent]] — agente Dockerizado (Nous Research).
- **Diretório**: =configs/hermes-workspace/=
- **Arquivos**: =docker-compose.yml= e configs do workspace.

** homepage-navi — Homepage Dashboard

- **Ferramenta**: [[https://github.com/benphelps/homepage][Homepage]] — dashboard de serviços self-hosted ("Navi").
- **Diretório**: =configs/homepage-navi/=
- **Arquivos**: =settings.yaml=, =services.yaml=, =bookmarks.yaml=, =widgets.yaml=, =status-server.py=.

** goose — Goose AI Agent (AAIF)

- **Ferramenta**: [[https://github.com/aaif-goose/goose][Goose]] — agente de IA open source (AAIF/Linux Foundation) com Desktop, CLI e API. Suporta 15+ providers via config declarativa e MCP.
- **Diretório**: =configs/goose/=
- **Arquivos**: =config.yaml= (config principal).
- **Provedor**: =ollama= (built-in — Ollama local na porta 11434, serve modelos =:cloud=). **Não usamos llama_swap** por causa de bug no canonical registry do Goose que renomeia modelos e causa 404.
- **MCP**: DuckDuckGo (via =uvx duckduckgo-mcp-server=) e SearXNG (via =npx -y mcp-searxng=).
- **Destino**: =~/.config/goose/config.yaml=
- **Sync**:
  #+BEGIN_SRC bash
  DOTFILES=~/git/ai-dotfiles
  cp "$DOTFILES/configs/goose/config.yaml" ~/.config/goose/config.yaml
  #+END_SRC

** opencode — OpenCode CLI

- **Ferramenta**: [[https://opencode.ai][OpenCode CLI]] — agente de código TUI com múltiplos agentes (build, fix, plan, review, debug, etc.).
- **Diretório**: =configs/opencode/=
- **Arquivos**: =opencode.json= (config principal), =tui.json= (tema TUI).
- **Provedores**: Ollama Cloud, llama.cpp local — modelos nuvem como default, modelos GGUF como alternativa.
- **MCP**: DuckDuckGo (via =uvx duckduckgo-mcp-server=) e SearXNG (via =npx -y mcp-searxng=).
- **Destino**: =~/.config/opencode/opencode.json= e =~/.config/opencode/tui.json=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/opencode/opencode.json ~/.config/opencode/opencode.json
  cp ~/git/ai-dotfiles/configs/opencode/tui.json ~/.config/opencode/tui.json
  #+END_SRC

** Modelos por Harness (resumo)

| Harness     | Regra local                     | Cloud | Obs                                     |
|-------------+---------------------------------+-------+-----------------------------------------|
| Pi          | Todos (22 base + 16 :think)     | 14    | Exceção: todos os modelos do fleet      |
| Crush       | Tools-only (14 base + 12 :think)| 15    | Sem modelos sem tool-calling            |
| Codex       | Tools-only (perfis TOML)        | 15    | Local limitado (Responses API)          |
| Droid       | Tools-only (14 base + 12 :think)| 15    | =noImageSupport= booleano               |
| OpenCode    | Tools-only (14 base + 12 :think)| 15    | =modalities= + =capabilities= schema    |
| Goose       | dynamic_models + lista explícita | 15+   | Providers declarativos JSON (SEM llama_swap — canonical registry bug) |

Notas:
- Modelos *always thinking* (GPT-OSS 20B) NÃO têm variante =:think=
- =gemma4-12b= é o único modelo local com vision+audio (=supports_attachments= / =input: [text, image, audio]=)
- O Codex local é limitado: llama-swap não suporta Responses API, usar =--oss --local-provider=ollama= como workaround
- Modelos de embedding (=nomic-embed-text-v2-moe=) e propósito específico (=hy-mt2-1.8b=, =translategemma-4b=, =webworld-8b=) só aparecem no Pi

* diffuse — Geração de Imagens Locais

[[https://prismml.com/news/bonsai-image-4b][Bonsai Image 4B]] rodando localmente na RTX 3050 6GB via gemlite + CUDA. Load-and-unload — gera imagem e libera VRAM.

- **Diretório**: =diffuse/=
- **CLI**: =diffuse generate -p "prompt"= (symlink em =~/.local/bin/diffuse=)
- **Comandos**: =diffuse setup=, =diffuse download=, =diffuse generate=, =diffuse evict=, =diffuse status=
- **Modelos**: binary-gemlite (1-bit), ternary-gemlite (1.58-bit, recomendado)
- **Workflow**: Evict LLM → load pipeline → generate → unload

Ver documentação completa em =diffuse/README.md=.

* magenta-rt — Geração de Música Local

[[https://github.com/magenta/magenta-realtime][Magenta RealTime 2]] rodando localmente na RTX 3050 via JAX + CUDA. Offline inference (streaming em tempo real requer Apple Silicon).

- **Diretório**: =magenta-rt/=
- **CLI**: =magenta-rt generate -p "disco funk"= (symlink em =~/.local/bin/magenta-rt=)
- **Comandos**: =magenta-rt setup=, =magenta-rt download=, =magenta-rt generate=, =magenta-rt evict=, =magenta-rt status=
- **Modelos**: mrt2_small (230M, recomendado RTX 3050), mrt2_base (2.4B)
- **Workflow**: Evict LLM → load model → generate audio → unload
- **Saída**: WAV 48kHz stereo

Ver documentação completa em =magenta-rt/README.md=.

* parakeet-stt — STT Local

[[https://github.com/mudler/parakeet.cpp][parakeet.cpp]] rodando nativamente com CUDA. NVIDIA Parakeet TDT 0.6B v3 (25 idiomas incluindo pt-BR).

- **Diretório**: =parakeet-stt/=
- **CLI**: =~/.local/bin/parakeet-stt <audio_file> --output_dir <tmp_dir>=
- **Modelo**: tdt-0.6b-v3 Q4_K (~644MB)
- **Integração Hermes**: =HERMES_LOCAL_STT_COMMAND= no =.env=

Ver documentação completa em =parakeet-stt/README.md=.

* llama-swap — Serviço de Modelos GGUF Locais

Configuração do [[https://github.com/nathan-nfx/llama-swap][llama-swap]] para servir múltiplos modelos GGUF com swap automático na RTX 3050 6GB.

- **Diretório**: =llama-swap/=
- **Arquivos**: =config.yaml= (config gerado — *não editar diretamente*), =config-base.yaml= + =config-footer.yaml= + =models/= (fontes fragmentadas), =build-config.py= (assembler), =download-models.sh= (script de download), =llama-swap-cli= (CLI helper), =testchat/= (script de teste).
- **Workflow**: Editar fragmentos em =models/= → =python3 build-config.py= → config regenerado e sincronizado automaticamente. Ver [[llama-swap/ARCHITECTURE.md][ARCHITECTURE.md]] para detalhes.
- **Dual-binary**: Usa dois backends — =llama.cpp= (upstream) para modelos densos e [[https://github.com/ikawrakow/ik_llama.cpp][ik_llama.cpp]] para modelos MoE, com =--fit= ativado e tamanhos de fit diferentes por backend (ik suporta contextos maiores para MoE).

** Comandos Principais

#+BEGIN_SRC bash
# Download de modelos
cd llama-swap
./download-models.sh qwopus-4b    # Modelo específico
./download-models.sh all           # Baixar todos

# Iniciar serviço
systemctl --user restart llama-swap
curl http://127.0.0.1:12434/v1/models
#+END_SRC

Ver documentação completa em =llama-swap/README.md=.
