* Sobre

Este repositório contém configurações /dotfiles/ para diversas ferramentas de IA — agentes de código, CLIs, dashboards e serviços Docker — todos versionados em um só lugar para facilitar sincronização entre máquinas.

* Estrutura do Repositório

#+BEGIN_SRC

├── configs/                    # Configurações por ferramenta
│   ├── ask-ai/                 # ask-ai-rs agent
│   ├── codex/                  # OpenAI Codex CLI
│   ├── crush/                  # Charm Crush CLI
│   ├── droid/                  # Factory.ai Droid
│   ├── hermes-workspace/       # Docker Compose para Hermes Agent
│   ├── homepage-navi/          # Homepage dashboard

│   ├── opencode-work/          # OpenCode CLI (máquina de trabalho)

│   ├── pi/                     # Pi Coding Agent (máquina pessoal)
│   ├── pi-work/                 # Pi Coding Agent (máquina de trabalho)

├── llama-swap/                 # Config do llama-swap + modelos GGUF

* Configurações por Ferramenta

Cada subdiretório de =configs/= é uma fonte de verdade (/source of truth/) para as configurações de uma ferramenta. Sincronize com =cp= ou =ln -s= conforme preferir.

** ask-ai — Agente ask-ai-rs

- **Ferramenta**: [[https://github.com/luksamuk/ask-ai-rs][ask-ai-rs]] — agente de IA no terminal com RAG e ferramentas MCP.
- **Diretório**: =configs/ask-ai/=
- **Sync**: Os configs são gerenciados internamente pelo =ask-ai-rs=; este diretório serve como backup/versionamento.

** codex — OpenAI Codex CLI

- **Ferramenta**: [[https://github.com/openai/codex][OpenAI Codex CLI]] — agente de código no terminal (Codex 0.128+).
- **Diretório**: =configs/codex/=
- **Arquivos**: =config.toml= — perfis de modelo (Ollama Cloud, llama-swap local, Sisyphus Pi), projetos confiáveis.
- **Destino**: =~/.codex/config.toml=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/codex/config.toml ~/.codex/config.toml
  #+END_SRC

** crush — Charm Crush CLI

- **Ferramenta**: [[https://charm.land/crush][Charm Crush]] — CLI de chat com IA, suporta múltiplos providers e MCP.
- **Diretório**: =configs/crush/=
- **Arquivos**: =crush.json= — providers (Ollama Cloud, llama.cpp local, Sisyphus Pi), modelos, MCP servers (DuckDuckGo via uvx, SearXNG), LSP, permissões.
- **Destino**: =~/.local/share/crush/crush.json=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/crush/crush.json ~/.local/share/crush/crush.json
  #+END_SRC

** droid — Factory.ai Droid

- **Ferramenta**: [[https://droid.factory.ai][Factory.ai Droid]] — agente de código com modelos customizados.
- **Diretório**: =configs/droid/=
- **Arquivos**: =settings.json= — modelos locais (llama.cpp), modelos nuvem (Ollama Cloud), modelos Pi.
- **Destino**: Diretório de config do Droid (verificar com =droid config=).

** hermes-workspace — Docker

- **Ferramenta**: [[https://hermes-agent.nousresearch.com][Hermes Agent]] — agente Dockerizado (Nous Research).
- **Diretório**: =configs/hermes-workspace/=
- **Arquivos**: =docker-compose.yml= e configs do workspace.
- **Uso**: =docker compose up= dentro do diretório.

** homepage-navi — Homepage Dashboard

- **Ferramenta**: [[https://github.com/benphelps/homepage][Homepage]] — dashboard de serviços self-hosted ("Navi").
- **Diretório**: =configs/homepage-navi/=
- **Arquivos**: =settings.yaml=, =services.yaml=, =bookmarks.yaml=, =widgets.yaml=, =status-server.py=.
- **Uso**: Copiar os YAMLs para o volume de config do container Docker do Homepage.

** opencode — OpenCode CLI (máquina pessoal)

- **Ferramenta**: [[https://opencode.ai][OpenCode CLI]] — agente de código TUI com múltiplos agentes (build, fix, plan, review, debug, etc.).
- **Diretório**: =configs/opencode/=
- **Arquivos**: =opencode.json= (config principal), =tui.json= (tema TUI).
- **Provedores**: Ollama Cloud, llama.cpp local — modelos nuvem como default, modelos GGUF como alternativa.
- **MCP**: DuckDuckGo (via =uvx duckduckgo-mcp-server=) e SearXNG (via =npx -y mcp-searxng=).
- **Destino**: =~/.config/opencode/opencode.json= e =~/.config/opencode/tui.json=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/opencode/opencode.json ~/.config/opencode/opencode.json
  cp ~/git/ai-dotfiles/configs/opencode/tui.json ~/.config/opencode/tui.json
  #+END_SRC

** opencode-work — OpenCode CLI (máquina de trabalho)

- **Ferramenta**: Variante do OpenCode para máquina de trabalho (IPs e providers diferentes).
- **Diretório**: =configs/opencode-work/=
- **Arquivos**: =opencode.jsonc= — usa Ollama em =192.168.3.50= e Sisyphus Pi como providers.
- **Destino**: =~/.config/opencode/opencode.jsonc=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/opencode-work/opencode.jsonc ~/.config/opencode/opencode.jsonc
  #+END_SRC

** open-webui — Open WebUI Docker

- **Ferramenta**: [[https://github.com/open-webui/open-webui][Open WebUI]] — interface web para modelos de IA (Docker).
- **Diretório**: =configs/open-webui/=
- **Uso**: Configs para bind-mount no container Docker do Open WebUI.

** pi — Pi Coding Agent (máquina pessoal)

- **Ferramenta**: [[https://pi.agent][Pi Coding Agent]] — agente de IA para programação, com suporte a MCP e extensões.
- **Diretório**: =configs/pi/=
- **Arquivos**: =settings.json= (config principal), =mcp.json= (MCP servers — DuckDuckGo e SearXNG), =models.json= (providers e modelos), =extensions/latency.ts=.
- **Destino**: =~/.pi/agent/=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/pi/settings.json ~/.pi/agent/settings.json
  cp ~/git/ai-dotfiles/configs/pi/mcp.json ~/.pi/agent/mcp.json
  cp ~/git/ai-dotfiles/configs/pi/models.json ~/.pi/agent/models.json
  #+END_SRC

- **Nota**: O arquivo =auth.json= (chaves de API) está no =.gitignore= e **não** é versionado.

** pi-work — Pi Coding Agent (máquina de trabalho)

- **Ferramenta**: Variante do Pi para máquina de trabalho (IPs e providers diferentes).
- **Diretório**: =configs/pi-work/=
- **Arquivos**: =settings.json=, =mcp.json=, =models.json= — usa Sisyphus Pi como default, modelos locais llama.cpp em =192.168.3.50=.
- **Destino**: =~/.pi/agent/=
- **Sync**:
  #+BEGIN_SRC bash
  cp ~/git/ai-dotfiles/configs/pi-work/settings.json ~/.pi/agent/settings.json
  cp ~/git/ai-dotfiles/configs/pi-work/mcp.json ~/.pi/agent/mcp.json
  cp ~/git/ai-dotfiles/configs/pi-work/models.json ~/.pi/agent/models.json
  #+END_SRC

** tui-studio — TUI Studio Docker

- **Ferramenta**: TUI Studio — ambiente Docker para desenvolvimento de TUI apps.
- **Diretório**: =configs/tui-studio/=
- **Uso**: Configs para bind-mount no container Docker.

* llama-swap — Serviço de Modelos GGUF Locais

Configuração do [[https://github.com/nathan-nfx/llama-swap][llama-swap]] para servir múltiplos modelos GGUF com swap automático na RTX 3050 6GB.

- **Diretório**: =llama-swap/=
- **Arquivos**: =config.yaml= (config principal), =download-models.sh= (script de download), =llama-swap-cli= (CLI helper), =testchat/= (script de teste).

** Comandos Principais

#+BEGIN_SRC bash
# Download de modelos
cd llama-swap
./download-models.sh qwopus-4b    # Modelo específico
./download-models.sh all           # Baixar todos

# Iniciar serviço
systemctl --user restart llama-swap
curl http://127.0.0.1:12434/v1/models
#+END_SRC

Ver documentação completa em =llama-swap/README.md=.

* Ollama Modelfiles

A pasta =modelfiles/= contém arquivos =.modelfile= e um =Makefile= para criar modelos especializados no Ollama.

#+BEGIN_SRC bash
# Criar todos os modelos essenciais
make -C modelfiles essentials

# Criar modelo específico
make -C modelfiles qwen3.5-4b

# Baixar modelos cloud
make -C modelfiles cloud-models
#+END_SRC

* scripts/ (DEPRECATED)

A pasta =scripts/= contém apenas o script =ask-ai.py=, que está **obsoleto** e não será mais mantido. Está presente apenas como registro histórico. O projeto [[https://github.com/luksamuk/ask-ai-rs][ask-ai-rs]] é a alternativa ativa.
