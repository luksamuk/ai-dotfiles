* Pi Coding Agent — Configs

Configurações para o [[https://github.com/badlogic/pi-mono][Pi Coding Agent]], assistente de código AI via CLI.

O Pi é um agente de codificação no terminal que suporta múltiplos providers (Ollama, llama.cpp/Serve, servers remotos), tool-calling, raciocínio (= :think =) e compação de contexto. Repositório: [[https://github.com/badlogic/pi-mono][github.com/badlogic/pi-mono]].

** Arquivos

| Arquivo              | Destino                        | Descrição                                  |
|----------------------+--------------------------------+--------------------------------------------|
| =models.json=        | =~/.pi/agent/models.json=      | Providers, modelos e compatibility flags   |
| =settings.json=      | =~/.pi/agent/settings.json=   | Modelo padrão, compação, pacotes           |
| =auth.json=          | =~/.pi/agent/auth.json=       | Credenciais — **NÃO commit**               |
| =mcp.json=           | =~/.pi/agent/mcp.json=        | Servidores MCP (DuckDuckGo, SearXNG)       |
| =extensions/latency= | =~/.pi/agent/extensions/=     | Extensão de medição de latência por modelo |

** Sync

Copiar do source of truth (ai-dotfiles) para o diretório runtime:

#+begin_src bash
# Configs principais
cp ~/git/ai-dotfiles/configs/pi/models.json   ~/.pi/agent/models.json
cp ~/git/ai-dotfiles/configs/pi/settings.json ~/.pi/agent/settings.json
cp ~/git/ai-dotfiles/configs/pi/mcp.json     ~/.pi/agent/mcp.json

# Extensão de latência (construir antes se necessário)
cp -r ~/git/ai-dotfiles/configs/pi/extensions ~/.pi/agent/extensions

# auth.json NÃO é sincronizado — contém secrets, gerenciar separadamente
#+end_src

** Configuração MCP

Servidores MCP configurados em =mcp.json=:

*** DuckDuckGo

#+begin_src json
{
  "duckduckgo": {
    "command": "uvx",
    "args": ["duckduckgo-mcp-server"],
    "lifecycle": "lazy",
    "idleTimeout": 10,
    "directTools": true
  }
}
#+end_src

- Usa =uvx duckduckgo-mcp-server= (instala via =pip install uvx=)
- =lifecycle: lazy= — sobe só quando necessário
- =idleTimeout: 10= — desliga após 10 min sem uso
- =directTools: true= — ferramentas disponíveis diretamente no Pi

*** SearXNG

#+begin_src json
{
  "searxng": {
    "command": "npx",
    "args": ["-y", "mcp-searxng"],
    "env": { "SEARXNG_URL": "http://192.168.3.8:8888" },
    "lifecycle": "lazy",
    "idleTimeout": 10
  }
}
#+end_src

- Usa =npx -y mcp-searxng= (executa sem instalação global)
- Aponta para instância SearXNG em =http://192.168.3.8:8888=
- =lifecycle: lazy= e =idleTimeout: 10=

** Compação de Contexto

Configuração recomendada em =settings.json= para modelos locais:

#+begin_src json
{
  "compaction": {
    "enabled": true,
    "reserveTokens": 32768,
    "keepRecentTokens": 15000
  }
}
#+end_src

- =reserveTokens: 32768= — reserva espaço suficiente para respostas longas (+ raciocínio em modelos =:think=)
- =keepRecentTokens: 15000= — mantém as mensagens mais recentes intactas durante compação
- Para modelos cloud com janela de 1M, pode-se aumentar =keepRecentTokens= para =30000= ou mais
- Modelo padrão atual: =qwen3.6-35b-opus-4.7:think= (provider =llama.cpp=)

** Uso

#+begin_src bash
# Usar modelo cloud
pi --model glm-5.1:cloud

# Usar modelo local (provider padrão já é llama.cpp)
pi --model qwen3.6-35b-moe

# Usar modelo local com raciocínio
pi --model "qwen3.6-35b-opus-4.7:think"

# Usar modelo edge no Sisyphus
pi --provider sisyphus --model qwen3-1.7b

# Listar todos os modelos disponíveis
pi --list-models
#+end_src

** Armadilhas (Pitfalls)

1. *O campo é =baseUrl=, não =baseURL=*. O Pi usa camelCase (=baseUrl=) diferente do padrão OpenAI (=baseURL=). Se usar =baseURL=, o provider não funciona.

2. *=models= deve ser um array*. Cada provider precisa de ="models": [...]= com objetos contendo pelo menos ="id"=, ="name"=, ="reasoning"=, ="contextWindow"= e ="maxTokens"=. String avulsa não funciona.

3. *=PATH= deve incluir =~/.hermes/node/bin=. O Pi instala o runtime Node.js em =~/.hermes/node/bin=. Se o =$PATH= não incluir esse diretório, extensões e servidores MCP via =npx= falham silenciosamente.

4. *Compatibilidade de tool-calling*. Modelos locais pequenos (< 7B) podem não suportar tool-calling corretamente. O Pi tenta adaptar, mas modelos como =qwen3-0.6b= e =gemma3-1b= no Sisyphus podem falhar em chamadas de ferramenta complexas.

5. *Não instalar via AUR*. O pacote =pi-coding-agent= no AUR está frequentemente desatualizado e pode quebrar configs. Instalar sempre via =npm= ou =pnpm= global.

6. *=auth.json= contém secrets* — **NUNCA** commitar no git. Manter fora do repositório.

7. *=llama.cpp= é o nome do provider* (não =local= nem =llama-swap=). Deve bater com a chave em =models.json=.

8. *=thinkingFormat: "qwen-chat-template"=* no provider =llama.cpp= é obrigatório para modelos =:think=. O llama.cpp requer =chat_template_kwargs.enable_thinking= no chat template.

9. *=supportsDeveloperRole: false= e =supportsReasoningEffort: false=* no provider =ollama= são necessários porque a API Ollama não implementa essas features do spec OpenAI.

10. *Modelos Gated Delta Net* (Qwen3.5, Gemma 4 E2B/E4B, etc.) têm visão desabilitada via =--no-mmproj= devido a bug CUDA no llama.cpp (issue #21394). Usar =input: ["text"]= em vez de =input: ["text", "image"]=.

11. *Pi recarrega =models.json= automaticamente* ao abrir o seletor de modelos (= /model = no TUI) — não é necessário reiniciar.

** Config de Trabalho (pi-work/)

Para máquinas de trabalho, criar um diretório =pi-work/= com overrides:

#+begin_src bash
mkdir -p ~/git/ai-dotfiles/configs/pi/pi-work
#+end_src

Estrutura sugerida:

#+begin_example
pi-work/
├── settings.json   # Modelo padrão diferente (ex.: cloud por padrão)
├── mcp.json        # MCP servers corporativos
└── auth.json       # Credenciais corporativas
#+end_example

Para sincronizar na máquina de trabalho:

#+begin_src bash
cp ~/git/ai-dotfiles/configs/pi/pi-work/settings.json ~/.pi/agent/settings.json
cp ~/git/ai-dotfiles/configs/pi/pi-work/mcp.json     ~/.pi/agent/mcp.json
# models.json pode ser o mesmo, ou adaptado para models restritos
#+end_src

O diretório =pi-work/= é versionado no git junto com o resto, mas =auth.json= de trabalho não deve ser commitado.
