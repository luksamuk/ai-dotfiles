* Pi Coding Agent Config

Config files for [[https://github.com/badlogic/pi-mono][Pi Coding Agent]] (CLI AI coding assistant).

** Files

| File             | Source                     | Description                      |
|------------------+----------------------------+----------------------------------|
| models.json      | ~/.pi/agent/models.json    | Provider and model definitions   |
| settings.json    | ~/.pi/agent/settings.json  | User settings (default model)    |
| auth.json        | ~/.pi/agent/auth.json      | Auth credentials (DO NOT commit) |

** Sync

Copy from source of truth (ai-dotfiles) to runtime location:

#+begin_src bash
cp ~/git/ai-dotfiles/configs/pi/models.json ~/.pi/agent/models.json
cp ~/git/ai-dotfiles/configs/pi/settings.json ~/.pi/agent/settings.json
# auth.json is NOT synced — contains secrets, manage separately
#+end_src

** Providers

- =ollama= — Cloud models via Ollama (porta 11434)
- =llama.cpp= — Local models via llama-swap (porta 12434)

** Usage

#+begin_src bash
# Use cloud model
pi --model glm-5.1:cloud

# Use local model (llama.cpp provider)
pi --provider llama.cpp --model qwen3.6-35b-moe

# Use local model with thinking
pi --provider llama.cpp --model "glm-4.7-flash:think"

# List all available models
pi --list-models
#+end_src

** Pitfalls

1. =auth.json= contains secrets — NEVER commit to git
2. Provider name is =llama.cpp= (not =local=) — matches llama-swap backend
3. The =llama.cpp= provider uses =thinkingFormat: "qwen-chat-template"= because llama.cpp needs =chat_template_kwargs.enable_thinking= for thinking mode
4. The =ollama= provider uses =supportsDeveloperRole: false= and =supportsReasoningEffort: false= because Ollama's API doesn't support these OpenAI-specific features
5. Default model is set in =settings.json= (currently =kimi-k2.6:cloud=)
6. =lastChangelogVersion= is set to current version to suppress changelog/update notices
7. Pi auto-reloads =models.json= when you open =/model= in the TUI — no restart needed
8. Gated Delta Net models (Qwen3.5, Gemma 4, GLM 4.7 Flash) have vision =disabled= due to llama.cpp CUDA crash bug (#21394)
