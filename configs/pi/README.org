* Pi Coding Agent Config

Config files for [[https://github.com/badlogic/pi-mono][Pi Coding Agent]] (CLI AI coding assistant).

** Files

| File             | Source                   | Description                      |
|------------------+--------------------------+----------------------------------|
| models.json      | ~/.pi/agent/models.json | Provider and model definitions   |
| settings.json    | ~/.pi/agent/settings.json | User settings (default model)   |
| auth.json        | ~/.pi/agent/auth.json    | Auth credentials (DO NOT commit) |

** Sync

Copy from source of truth (ai-dotfiles) to runtime location:

#+begin_src bash
cp ~/git/ai-dotfiles/configs/pi/models.json ~/.pi/agent/models.json
cp ~/git/ai-dotfiles/configs/pi/settings.json ~/.pi/agent/settings.json
# auth.json is NOT synced — contains secrets, manage separately
#+end_src

** Usage

#+begin_src bash
# Use cloud model
pi --model glm-5.1:cloud

# Use local Qwen3.6 27B (no thinking)
pi --model local/qwen3.6-27b

# Use local Qwen3.6 27B Thinking
pi --model local/qwen3.6-27b-think

# List all available models
pi --list-models
#+end_src

** Pitfalls

1. =auth.json= contains secrets — NEVER commit to git
2. The =local= provider uses =thinkingFormat: "qwen-chat-template"= because llama-swap/llama.cpp needs =chat_template_kwargs.enable_thinking= for thinking mode
3. The =ollama= provider uses =supportsDeveloperRole: false= and =supportsReasoningEffort: false= because Ollama's API doesn't support these OpenAI-specific features
4. Default model is set in =settings.json= (currently =kimi-k2.6:cloud=)
5. Pi auto-reloads =models.json= when you open =/model= in the TUI — no restart needed
