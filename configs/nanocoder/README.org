#+TITLE: Nanocoder configuration

Nanocoder (https://github.com/Nano-Collective/nanocoder) is a community-built
terminal coding agent. This directory holds its configuration.

- ~agents.config.json~: Full provider config — Ollama Cloud + llama-swap
  local, context windows, auto-compact, and session settings.
- ~/.mcp.json~: MCP servers — SearXNG meta-search engine.

* Sync

#+BEGIN_SRC sh
cp ~/git/ai-dotfiles/configs/nanocoder/agents.config.json ~/.config/nanocoder/agents.config.json
cp ~/git/ai-dotfiles/configs/nanocoder/.mcp.json ~/.config/nanocoder/.mcp.json
#+END_SRC

Or use ~/setup-providers~ inside Nanocoder for a guided wizard.

* MCP servers

Nanocoder loads MCP servers from =~/.config/nanocoder/.mcp.json= (global)
and =.mcp.json= in the project root (project-level). Project-level takes
precedence when the same server name exists in both.

** Configured servers

| Server  | Transport | Description                                                |
|---------+-----------+------------------------------------------------------------|
| searxng | stdio     | Meta-search via local SearXNG at http://192.168.3.8:8888 |

SearXNG aggregates Google, Bing, DuckDuckGo, Wikipedia and dozens more
engines. No API key needed (self-hosted on the RPi).

** Adding new servers

Edit =~/.config/nanocoder/.mcp.json= or the project-level =.mcp.json=.
Use =/setup-mcp= inside Nanocoder for an interactive wizard, or =/mcp=
to view connected servers and their tools.

** Sync from ai-dotfiles

#+BEGIN_SRC sh
cp ~/git/ai-dotfiles/configs/nanocoder/.mcp.json ~/.config/nanocoder/.mcp.json
#+END_SRC

* Config format differences (nanocoder vs opencode)

| Feature                | opencode (=opencode.json=)           | nanocoder (=agents.config.json=)              |
|------------------------+--------------------------------------+------------------------------------------------|
| Top-level key          | =provider=, =agent=, =mcp=           | =nanocoder.providers=, =nanocoder.autoCompact= |
| Provider structure     | =provider.<name>.models.<id>={...}=   | =providers[].name + models[]=                 |
| Context windows        | =limit.context= per model             | =contextWindows= dict on provider              |
| Model capabilities     | =capabilities: ["tools"]=             | =disableTools= / =disableToolModels=           |
| Vision support         | =modalities= per model                | Not declared in config; models.dev auto-resolve|
| SDK providers          | =enabled_providers= + =npm=            | =sdkProvider= per provider                     |
| Agent roles            | =agent.<role>.model=                  | Not in config (=systemPrompt= + =alwaysAllow=) |
| MCP servers             | =mcp.<name>={...}=                    | =mcpServers=[{...}]=                           |

* Key nanocoder config options

** Auto-compact

Configured at threshold 70% with LLM strategy (conservative mode). This
means nanocoder automatically compresses context when it reaches 70% of the
model's context window, using the model itself to write a structured summary.

** Session management

Auto-save enabled with 30-second intervals, max 100 sessions retained for
30 days.

** Timeouts

Both cloud and local providers use =requestTimeout: -1= and =socketTimeout: -1=
(disabled) to handle long inference times on local models.

* Notes on =:think= variants

llama-swap supports =:think= variants via =setParamsByID=, which enables
thinking/reasoning mode on models that support it. In nanocoder, these appear
as separate model names. They inherit the same context window as their base
model — the =contextWindows= map explicitly sets both to the same value for
clarity.
